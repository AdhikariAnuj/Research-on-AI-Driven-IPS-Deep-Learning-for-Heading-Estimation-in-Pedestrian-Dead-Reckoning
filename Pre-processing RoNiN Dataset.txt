The original split RONIN dataset was used: train_dataset_1 and train_dataset_2 were used together for training, seen_subjects_test_set was used for validation, and unseen_subjects_test_set was saved for the final evaluation. This split was kept the way it was originally planned by the authors to make sure that the test set has subjects that were never seen during training. This is done to make sure that the generalization test is accurate for unseen individuals.
Three inertial sensors—an accelerometer, a gyroscope, and a magnetometer—each provide x, y, and z axis measures, resulting in a 9-channel input vector per timestep. These readings are time-synchronized and sampled at 200Hz in each sequence file.
There was no ground truth heading directly in the dataset. Instead, it was inferred from the tango_pos position coordinates. At each timestep, the arctangent function — arctan2(dy, dx) — was used to find the direction of movement between two consecutive x/y values. This gave a continuous heading angle in radians. Figure  shows that when we write the raw heading in degrees, there are sharp breaks at the ±180° wraparound boundary. This is because a physically small change in direction causes a numerically big jump. If we used raw degrees as a training target, the loss function would penalize right predictions close to this boundary in an incorrect manner. To fix this, heading was broken down into two continuous output values, sin(heading) and cos(heading), which got rid of the discontinuity completely. At evaluation time, heading in degrees is found using arctan2(sin, cos). The change is lossless as the reconstruction error is less than 1e-14 degrees.
The raw sensor data was split into fixed-length windows of 200 timesteps, which is equal to one second of data at 200Hz. This was done using a sliding window method with a step size of 50 timesteps, which made the windows overlap by 75%. The heading number at the last timestep of each window is the label that is given to that window. It takes in shapes (200, 9) and sends out shapes (2,). There were 82,656 windows in the confirmation set. Once the whole dataset has been processed, the final counts for the training set and the test set will be shown in the results section
Each of the 9 input channels got its own Z-score normalization process. The training set was used to only calculate the mean and standard deviation, which were then used on the validation and test sets. This made sure that information from the held-out data did not change the normalization values.
